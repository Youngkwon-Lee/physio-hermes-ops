# BRIEFING_FORMAT_SPEC

문서 목적: physio-hermes-ops에서 운영하는 Hermes 브리핑/다이제스트/킥오프 메시지의 **품질 기준**을 고정한다.  
이 문서는 사용자-facing output의 구조 일관성과 실사용성을 유지하기 위한 spec이다.

---

## 1. 공통 원칙

모든 브리핑은 다음 원칙을 따른다.

1. **짧게**  
   길게 설명하기보다 빠르게 훑고 행동할 수 있어야 한다.

2. **구조적으로**  
   핵심 요약 → 항목별 정리 → 다음 액션 순서가 유지되어야 한다.

3. **근거 기반으로**  
   추측을 줄이고, 실제 검색/세션/스크립트/원문 근거가 있는 내용만 쓴다.

4. **영권님 관점으로 재해석**  
   일반 요약이 아니라 재활, 임상 AI, physio_app, 에이전트 운영, 지식관리 관점에서 왜 중요한지 연결한다.

5. **행동 가능하게**  
   가능한 경우 반드시 “오늘 바로 볼 것”, “첫 액션”, “체크포인트” 중 하나를 포함한다.

6. **과장 금지**  
   큰 신호가 없으면 없다고 명시한다. 억지 인사이트를 만들지 않는다.

---

## 2. 브리핑 타입

### A. News Briefing
예: daily-ai-news-briefing, daily-rehab-ai-research-brief

**목적**  
최근 정보 중 실제로 볼 가치가 있는 항목만 선별해 전달한다.

**권장 형식**

```md
# 제목
- 핵심 3줄

- TOP N
  - 제목 | 한줄 요약 | 왜 중요한지 | 링크

- 영권님 관점 메모
  - 1~3개

- 오늘 바로 볼 것
  - 1~3개
```

**품질 기준**
- 핵심 3줄은 전체 방향을 먼저 보여줘야 한다.
- TOP N은 3~5개 권장.
- 링크는 가능한 원문 링크 우선.
- “왜 중요한지”는 일반론보다 실제 활용 관점이 좋다.

---

### B. Discussion Kickoff
예: daily-ai-news-discussion-kickoff

**목적**  
브리핑 내용을 반복 요약하는 것이 아니라, 오늘 판단해야 할 질문과 실험 포인트를 만든다.

**권장 형식**

```md
# 제목
- 오늘 뉴스 한줄 요약

- 오늘 생각할 질문 3개
  - 질문

- 실험/추적 체크포인트 3개
  - 체크포인트

- 넘겨도 되는 항목 1~2개
  - 항목
```

**품질 기준**
- 질문은 전략/실행 중심이어야 한다.
- 브리핑 원문에 없는 사실을 만들지 않는다.
- 억지 통찰보다 “오늘 무엇을 확인할지”가 중요하다.

---

### C. Operational Morning Brief
예: daily-calendar-mail-brief, home-rehab-morning-brief

**목적**  
하루 일정/메일/운영 우선순위를 짧고 명확하게 보여준다.

**권장 형식**

```md
# 제목
## 오늘 일정
- 시간 | 대상 | 유형 | 위치

## 내일 일정
- ...

## 최근 중요 항목
- 중요 메일 / 경고 / 체크포인트

## 오늘의 할 일
- 1~3개

## 우선순위 액션
- P1/P2/P3 형태
```

**품질 기준**
- 일정은 스캔하기 쉬운 포맷이어야 한다.
- 개인 정보는 최소화한다.
- 메일은 raw dump가 아니라 우선순위만 보여준다.
- 마지막엔 반드시 행동 우선순위를 남긴다.

---

### D. Overnight / Handoff Summary
예: overnight-pt-morning-summary

**목적**  
밤사이 논의/리서치/운영 흐름을 아침에 바로 이어볼 수 있게 만든다.

**권장 형식**

```md
# 제목
- 밤사이 핵심 3줄

- 이어볼 것 3개

- 보류/리스크 1~3개

- 오늘 오전 첫 액션 1개
```

**품질 기준**
- 시간 범위를 엄격히 지킨다.
- 각 항목은 실제 세션/브리핑 근거가 있어야 한다.
- 가능하면 근거 시간대 또는 출처를 짧게 암시한다.
- 내용이 약하면 `[SILENT]`를 허용한다.

---

### E. Digest / Curation
예: daily-discord-digest, daily-conversation-curator

**목적**  
대화 로그 전체가 아니라 장기적으로 의미 있는 결정/논점/패턴만 남긴다.

**권장 형식**

```md
# 제목
- 오늘의 핵심 요약

- 주요 결정
  - 결정 / 근거 / 영향

- 보류된 논점
  - 항목

- 장기 기억 후보
  - 후보 / 이유
```

**품질 기준**
- 단순 대화 요약이 아니라 “나중에 다시 찾을 가치”가 있는 것만 남긴다.
- 임시 TODO, 일회성 artifact, 금방 stale해질 정보는 제외한다.

---

## 3. 금지 패턴

다음 패턴은 피한다.

- 단순 나열만 있고 우선순위가 없는 출력
- 링크만 있고 왜 중요한지 설명이 없는 출력
- 실제 근거 없이 확신하는 문장
- 일반론으로 부풀린 인사이트
- 너무 많은 항목을 담아 스캔이 어려운 출력
- private/raw 데이터를 그대로 재노출하는 출력

---

## 4. 좋은 브리핑 체크리스트

브리핑 작성 후 아래를 점검한다.

- 첫 3줄만 읽어도 전체 맥락이 보이는가?
- 사용자가 “그래서 오늘 뭘 보면 되지?”를 바로 알 수 있는가?
- 항목 수가 과하지 않은가?
- 링크/세션/스크립트 근거가 있는가?
- 영권님 관점의 해석이 실제로 도움이 되는가?
- 불확실한 부분을 솔직하게 표현했는가?

---

## 5. 운영 메모

- runtime output은 Hermes cron state가 진실 소스다.
- 이 문서는 **출력 형식과 품질 기준**을 정의하는 레벨이다.
- 프롬프트를 수정할 때는 이 spec과 어긋나지 않는지 먼저 확인한다.
